醜陋的 Option/Result 處理

當你只是試圖快速地寫一些代碼時，Option/Result 對付起來可能比較無聊。 Option 和 Result 都有一個名為 unwrap 的函數：這個函數可以簡單粗暴地獲取其中的值。 unwrap 會：

  1. 獲取 Option/Result 內部的值
  2. 如果枚舉的類型是 None/Err， 則會 panic!

這兩段代碼是等價的：

my_option.unwrap()

match my_option {
    Some(v) => v,
    None => panic!("some error message generated by Rust!"),
}

類似的：

my_result.unwrap()

match my_result {
    Ok(v) => v,
    Err(e) => panic!("some error message generated by Rust!"),
}

不過啊，做個好 Rustacean，正確地使用 match！
